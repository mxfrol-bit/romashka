# ПЛАН МИГРАЦИИ НА ОБЛАЧНЫЕ ТЕЛЕФОНЫ (VMOS Cloud)

## Цель
Полностью заменить физическую ферму на облачные Android-устройства через VMOS Cloud (или аналог) с сохранением всей текущей логики проекта.

## Этап 1: Подготовка (1–2 дня)

### 1.1 Регистрация и тестирование
- [ ] Зарегистрироваться в VMOS Cloud
- [ ] Создать тестовый аккаунт и пополнить баланс
- [ ] Создать 3–5 тестовых облачных устройств
- [ ] Протестировать базовые действия через веб-интерфейс:
  - Запуск/остановка устройства
  - Установка APK
  - Запуск приложений
  - Ввод текста и клики

### 1.2 Получение доступа к API
- [ ] Запросить доступ к Open API у поддержки VMOS (если требуется)
- [ ] Получить API Token
- [ ] Изучить документацию:
  - https://www.vmoscloud.com/open-api
  - Android Control API

### 1.3 Тестирование API
- [ ] Сделать тестовые запросы через Postman / curl:
  - Получить список устройств
  - Создать устройство
  - Запустить устройство
  - Выполнить простую команду (например, открыть приложение)

## Этап 2: Интеграция в проект (2–3 дня)

### 2.1 Создать модуль работы с облачными устройствами
- [ ] Создать папку `cloud_devices/`
- [ ] Создать файл `vmos.py` с классом `VMOSCloudProvider`
- [ ] Реализовать базовые методы:
  - `list_devices()`
  - `create_device(name, image_id)`
  - `start_device(device_id)`
  - `stop_device(device_id)`
  - `get_device_status(device_id)`
  - `execute_command(device_id, command)` — для Android Control API

### 2.2 Обновить систему чекеров
- [ ] Добавить возможность запускать чекеры (truepeople, beenverified и т.д.) на облачных устройствах
- [ ] Создать абстракцию `DeviceProvider` (чтобы можно было легко переключаться между физической фермой и облаком)

### 2.3 Обновить пайплайн
- [ ] Создать новый скрипт `cloud_farm_runner.py`
- [ ] Или обновить `automation/full_pipeline.py`, добавив поддержку облачных устройств
- [ ] Сделать выбор: физическая ферма / облачная ферма

### 2.4 Обновить веб-интерфейс
- [ ] В Streamlit (`web_app.py`) добавить переключатель "Физическая ферма / Облачная ферма"
- [ ] Добавить раздел управления облачными устройствами (список, запуск, остановка)

## Этап 3: Миграция и тестирование (3–5 дней)

### 3.1 Тестирование на малом объёме
- [ ] Взять 10–20 номеров
- [ ] Запустить полный цикл через облачные устройства:
  1. Проверка истории номера (5 чекеров)
  2. Фильтрация
  3. "Покупка" / активация на облачном устройстве
  4. Регистрация / апрув аккаунта

### 3.2 Настройка массовых операций
- [ ] Настроить массовый запуск устройств
- [ ] Настроить синхронизацию действий (Synchronizer в VMOS)
- [ ] Добавить логирование и мониторинг

### 3.3 Мониторинг и алерты
- [ ] Добавить логирование всех действий
- [ ] Настроить уведомления при ошибках (Telegram / Email)

## Этап 4: Полная миграция (1 неделя)

- [ ] Перенести все активные устройства на облако
- [ ] Отключить физическую ферму
- [ ] Обновить всю документацию
- [ ] Обучить команду работе с новым интерфейсом

## Этап 5: Оптимизация и масштабирование

- [ ] Настроить автоматическое создание/удаление устройств по необходимости
- [ ] Добавить балансировку нагрузки между устройствами
- [ ] Внедрить AI Agent от VMOS для сложных сценариев
- [ ] Настроить резервное копирование и восстановление устройств

---

## Рекомендуемый стек после миграции

- **Backend**: FastAPI (уже есть)
- **Frontend**: Streamlit + HTML (уже есть)
- **Облачные устройства**: VMOS Cloud
- **Автоматизация**: Комбинация VMOS API + Android Control API + Python скрипты
- **Оркестрация**: Локальный агент или облачный worker
- **Мониторинг**: Логи + Telegram-бот

## Риски и как их минимизировать

| Риск                        | Вероятность | Как минимизировать                          |
|----------------------------|-------------|---------------------------------------------|
| API VMOS изменится         | Средняя     | Использовать версионирование + обёртки     |
| Устройства будут банить    | Средняя     | Хорошие прокси + качественные отпечатки    |
| Высокая стоимость          | Высокая     | Начинать с малого количества устройств     |
| Сложность отладки          | Высокая     | Вести подробные логи + скриншоты           |
